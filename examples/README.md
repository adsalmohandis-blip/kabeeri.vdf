# Examples

This directory contains practical example workflows for **Kabeeri Vibe Developer Framework**.

The current examples folder started with three placeholder folders:

```text
examples/lite/
examples/standard/
examples/enterprise/
```

This improved version turns them into the first usable example set for `v0.1.1`.

## Purpose

Examples help new users understand how to use the framework from idea to AI-assisted implementation.

They show how a vibe developer moves through:

```text
Raw idea
→ project profile
→ questions
→ generated planning documents
→ prompt pack selection
→ task tracking
→ acceptance review
→ GitHub issue closure
```

## What this folder is

This folder provides reference examples for the three generator levels:

```text
lite
standard
enterprise
```

Each example explains:

- when to use this profile
- what kind of product fits it
- what folders/documents are expected
- what questions are answered
- what prompt pack can be used
- what task tracking looks like
- what acceptance checklist should be applied

## What this folder is not

This folder is not a complete production application.

It does not include real application code, real credentials, deployment configuration, or private customer data.

It is a learning and reference layer for Kabeeri VDF users and contributors.

## Example profiles

| Profile | Best for | Complexity |
|---|---|---|
| Lite | Simple MVPs and small tools | Low |
| Standard | Real products, SaaS MVPs, client projects | Medium |
| Enterprise | Larger systems, teams, multi-role platforms | High |

## Recommended starting point

New users should start with:

```text
examples/lite/
```

Then move to:

```text
examples/standard/
```

Use enterprise only when the product truly needs complex structure, roles, governance, and release controls.
