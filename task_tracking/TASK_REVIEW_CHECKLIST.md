# Task Review Checklist

Use this checklist before moving a task to Done or closing a GitHub Issue.

## Scope review

```text
- [ ] The task stayed inside the original goal.
- [ ] No future features were added.
- [ ] No unrelated folders were modified.
- [ ] The result matches the issue description.
```

## File review

```text
- [ ] File names are clear.
- [ ] Files are in the correct folder.
- [ ] No duplicate files were created accidentally.
- [ ] No generated temporary files were committed.
```

## Content review

```text
- [ ] The content is beginner-friendly.
- [ ] Instructions are direct and practical.
- [ ] Technical language is explained when needed.
- [ ] AI assistant instructions are clear.
- [ ] Acceptance criteria are measurable.
```

## Safety review

```text
- [ ] No real secrets were added.
- [ ] No private keys were added.
- [ ] No personal data was added.
- [ ] No production credentials were added.
```

## GitHub review

```text
- [ ] The issue has the correct labels.
- [ ] The issue is assigned to the correct milestone.
- [ ] The project card is moved correctly.
- [ ] The commit message is clear.
- [ ] The issue can be closed.
```

## Acceptance decision

```text
accepted
```
