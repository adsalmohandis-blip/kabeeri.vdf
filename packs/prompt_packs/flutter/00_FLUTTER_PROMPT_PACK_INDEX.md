# 00 — Flutter Prompt Pack Index

## Framework

Kabeeri Vibe Developer Framework

## Prompt pack

Flutter

## Version

v0.1.1

## Goal

Provide a safe, beginner-friendly Flutter prompt pack that helps vibe developers build a mobile app step by step after completing Kabeeri VDF planning documents.

## What this pack is

This pack is a set of structured prompts for AI coding assistants.

It helps AI work on a Flutter project in a controlled order.

## What this pack is not

This pack is not:

- a Flutter installer
- a Dart installer
- an Android Studio or Xcode setup guide
- a mobile app by itself
- a replacement for official Flutter documentation

## Before using this pack

The user should already have, or should ask AI to help create:

```text
01_STRATEGY_AND_BUSINESS
02_FOUNDATION_ARCHITECTURE
03_RELEASE_SPECS
04_DATABASE_ARCHITECTURE or API/data plan
05_EXECUTION_PLAN
06_ENGINEERING_STANDARDS
08_AI_CODE_PROMPTS
09_TASK_TRACKING
10_ACCEPTANCE_CHECKLISTS
```

For Lite mobile apps, not all folders are required, but the first release scope should be clear.

## Prompt order

Use the prompts in this order:

```text
01_MOBILE_PROJECT_CONTEXT_PROMPT.md
02_PROJECT_STRUCTURE_NAVIGATION_PROMPT.md
03_ENV_CONFIG_API_PROMPT.md
04_THEME_DESIGN_SYSTEM_PROMPT.md
05_AUTH_ONBOARDING_PROMPT.md
06_API_DATA_STATE_PROMPT.md
07_CORE_SCREENS_FEATURES_PROMPT.md
08_FORMS_VALIDATION_PROMPT.md
09_LOCAL_STORAGE_OFFLINE_PROMPT.md
10_DEVICE_PERMISSIONS_NOTIFICATIONS_PROMPT.md
11_TESTING_REVIEW_PROMPT.md
12_RELEASE_HANDOFF_PROMPT.md
```

## What each prompt does

| Prompt | Purpose |
|---|---|
| 01 | Give AI the product and mobile app context. |
| 02 | Review or prepare Flutter structure and navigation. |
| 03 | Configure environment, API URL, and app config rules. |
| 04 | Create theme, design system, and reusable widget foundation. |
| 05 | Add authentication and onboarding foundation if needed. |
| 06 | Add API, data fetching, and state management foundation. |
| 07 | Add first product-specific screens and features. |
| 08 | Add forms and validation foundation. |
| 09 | Add local storage and offline foundation if needed. |
| 10 | Add device permissions and push notification foundation if needed. |
| 11 | Review tests, device checks, accessibility, and quality. |
| 12 | Prepare release handoff summary. |

## Important

These prompts are intentionally small. They should not be merged into one large prompt.
