# Security Policy

This framework generates project structures, documents, prompts, and execution workflows. It may influence how users build real software, so security guidance matters.

## Report security issues

Please report security issues privately to the maintainers. Do not open public issues for vulnerabilities that could be exploited.

## Security principles

- Do not store secrets in generated documents.
- Generated prompts must instruct AI not to expose API keys, passwords, or private tokens.
- Prompt packs must include validation, authorization, logging, and error-handling guidance.
- Any integration template must include secure credential handling.
- Generated code must be reviewed by a human before production use.

## Out of scope

This repository does not guarantee security of applications generated by third-party AI tools. It provides structure and guidance only.
