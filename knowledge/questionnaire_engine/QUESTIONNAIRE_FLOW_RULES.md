# Questionnaire Flow Rules

- Start with entry questions only.
- Detect project type, complexity, users, admin, payments, tenancy, public frontend, integrations, and AI feature needs.
- Activate system areas as `required`, `optional`, `deferred`, `not_applicable`, `unknown`, or `needs_follow_up`.
- Never silently skip an area unless it is answered, deferred, or not applicable.
- Every generated task must keep provenance fields for system area, question IDs, answer IDs, and source mode.
