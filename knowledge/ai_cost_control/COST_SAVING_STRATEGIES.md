# Cost Saving Strategies

## Before AI Execution

- Use existing templates and schemas.
- Read the task and acceptance criteria first.
- Generate a context pack instead of sending the whole repository.
- Summarize large files once, then reuse the summary.
- Split broad work into smaller tasks.
- Run local tests and validators before asking AI to diagnose.

## During AI Execution

- Keep prompts scoped to the task.
- Include allowed and forbidden files.
- Ask for minimal diffs or focused analysis.
- Avoid repeated full-context retries.
- Stop when the output is clearly off-track and refine the task.

## After AI Execution

- Record usage with task, workstream, provider, model, result, and cost.
- Classify rejected and rework cost.
- Add useful findings to project memory.
- Update estimation rules from actual cost.

## Team Habits

- Review high-cost tasks in planning.
- Prefer small acceptance criteria over vague large requests.
- Keep ADRs current so AI does not rediscover decisions.
- Use low-cost mode for routine docs, labels, and summaries.
