# Sprint Cost Control

Sprint cost control makes Agile AI usage visible before, during, and after each sprint.

## Sprint Planning

During sprint planning:

- estimate token cost for each story
- identify high-cost stories
- split broad stories
- reserve exploration budget
- define sprint warning thresholds
- choose model routing defaults

## During Sprint

Track:

- tokens used by story and task
- cost by workstream
- cost by developer or AI agent
- accepted vs rejected output
- rework cost
- untracked usage
- budget warnings

## Sprint Review

Sprint review should include:

- planned vs actual token cost
- cost per accepted story
- rework percentage
- untracked cost explanation
- lessons for next sprint estimation

## Rule

A sprint can finish with untracked usage, but it cannot hide it.
