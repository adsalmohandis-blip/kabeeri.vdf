# System Capability Map

Kabeeri maps project type to system areas, then uses answers to decide whether each area is required, optional, deferred, not applicable, unknown, or needs follow-up.

The same catalog is also exposed as a capability registry in
[`CAPABILITY_REGISTRY.md`](CAPABILITY_REGISTRY.md) and through
`kvdf capability registry`.

The ten capability groups are:

- Product & Business
- Users, Access, and Journeys
- Frontend Experience
- Backend, Data, and APIs
- Admin, Settings, and Customization
- Engagement, Content, and Growth
- Commerce and Integrations
- Quality, Security, and Compliance
- Operations and Release
- Kabeeri Control Layer
