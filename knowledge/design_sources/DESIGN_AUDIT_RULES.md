# Design Audit Rules

Design audit checks whether frontend work is safe to implement.

## Required Checks

- approved design source exists
- approved text spec exists
- design tokens exist or are intentionally deferred
- page spec exists for each page
- component contract exists for repeated components
- loading, empty, error, success, and disabled states are defined where applicable
- responsive behavior is defined
- accessibility requirements are defined
- allowed and forbidden files are listed
- visual acceptance checklist is ready

## Blockers

Implementation is blocked when a page has only a raw image, PDF, link, or reference website with no approved text spec.
