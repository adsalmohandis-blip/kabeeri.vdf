# Design Source To Text Spec Rules

## Rule

Frontend implementation may begin only after the design source has been converted into an approved text spec.

## Required Text Spec Sections

- source ID and snapshot reference
- source type
- extraction mode
- page or component scope
- layout structure
- content requirements
- design token references
- responsive behavior
- component states
- data requirements
- accessibility notes
- open questions
- approval status

## Draft Status

Any spec extracted from a visual source is `draft` until Owner, client, designer, or assigned reviewer approval.

## No Silent Filling

Missing design details must be recorded in `MISSING_DESIGN_REPORT_TEMPLATE.md`. AI must not silently invent colors, typography, spacing, states, or identity.
