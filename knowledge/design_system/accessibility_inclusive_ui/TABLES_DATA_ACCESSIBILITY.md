# Tables And Data Accessibility

## Data Tables

- Tables need headers that describe each column.
- Complex tables need captions or summaries.
- Sortable headers expose sort state.
- Row actions must include the row subject in accessible names when the visible label is generic.
- Pagination communicates current page, total pages when known, and disabled states.
- Filters expose active filter state and clear filter actions.
- Empty and error states must be reachable and understandable to screen readers.

## Dense Operational Screens

- Provide column visibility or responsive alternatives when tables become too wide.
- Do not hide critical data behind hover-only UI.
- Bulk actions need selection count and confirmation for destructive operations.
- Audit, permission, and status information should not rely on color alone.

## Charts

- Charts need text summaries or data tables for key insights.
- Color series need labels or patterns when meaning matters.
- Tooltips should not be the only way to access chart values.
