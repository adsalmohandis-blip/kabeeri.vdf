# Accessibility And Inclusive UI Reference Pack

Use this pack for every UI implementation, refactor, review, or design recommendation.

The goal is to make accessibility a default engineering constraint, not a late visual polish step. Every interface should remain usable with keyboard, screen reader, zoom, reduced motion, high contrast needs, clear error recovery, and understandable content.

Core order:

1. Choose accessibility level for the product risk and surface type.
2. Apply `ACCESSIBILITY_CONTRACT.md`.
3. Apply focus, keyboard, semantic HTML, forms, tables, dialogs, and content rules.
4. Apply motion, contrast, touch target, and responsive accessibility rules.
5. Run `ACCESSIBILITY_REVIEW_CHECKLIST.md` before delivery.

Files:

- `ACCESSIBILITY_CONTRACT.md`
- `FOCUS_KEYBOARD_RULES.md`
- `SEMANTIC_HTML_RULES.md`
- `FORMS_ERRORS_RULES.md`
- `TABLES_DATA_ACCESSIBILITY.md`
- `DIALOGS_OVERLAYS_RULES.md`
- `CONTENT_READABILITY_RULES.md`
- `ACCESSIBILITY_REVIEW_CHECKLIST.md`
