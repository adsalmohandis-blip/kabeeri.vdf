# Content Readability Rules

## Text

- Use clear headings and short sections.
- Avoid long dense paragraphs in product UI.
- Keep line length comfortable for content pages.
- Use plain action labels that describe outcome.
- Error copy should explain what happened and how to recover.
- Empty states should state what is missing and the next useful action.

## Visual Support

- Icons support meaning but do not replace labels for important actions.
- Images need alt text when informative and empty alt when decorative.
- Badges and statuses include text, not color alone.
- Skeletons should not create layout jumps.

## International And Cognitive Load

- Avoid idioms in critical workflows.
- Keep dates, numbers, currency, and units explicit.
- Progressive disclosure is preferred for long settings or forms.
- Confirmation language should be direct for irreversible actions.
