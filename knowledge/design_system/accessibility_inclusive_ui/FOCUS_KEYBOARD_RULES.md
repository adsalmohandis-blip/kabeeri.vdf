# Focus And Keyboard Rules

## Focus

- Every actionable control must have a visible focus state.
- Focus order should follow visual and reading order.
- Do not remove outlines unless a replacement focus style is defined.
- Focus rings should be visible against light, dark, and brand surfaces.
- After form submission failure, focus should move to the summary or first invalid field.
- After modal close, focus returns to the control that opened it.
- After route or major view change, focus should move to the page heading or main region.

## Keyboard

- Buttons activate with Enter and Space.
- Links navigate with Enter.
- Menus, dropdowns, tabs, accordions, and dialogs follow expected keyboard behavior.
- Escape closes transient overlays when safe.
- Keyboard users can complete all primary flows without pointer input.
- Keyboard shortcuts must not block browser or assistive technology shortcuts.
- Icon-only toolbar controls need accessible names and tooltips when helpful.

## Review Prompts

- Can the complete task be performed with keyboard only?
- Is focus always visible?
- Does focus move predictably after modal, validation, and navigation events?
- Are custom controls justified and fully keyboard accessible?
