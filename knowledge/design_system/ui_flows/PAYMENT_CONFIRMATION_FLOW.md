# Payment Confirmation Flow

Use for any financial, subscription, invoice, transfer, or paid action.

Steps:

1. Review amount, recipient, item, plan, or invoice.
2. Confirm intent with a clear primary action.
3. Show processing without implying completion.
4. Show success or failure with receipt, retry, or support path.

Required states:

- Review
- Confirm
- Processing
- Success
- Failure
- Receipt

Rules:

- No confetti for serious financial actions.
- Amounts remain readable and stable.
- Use minimal motion and strong static confirmations.
