# Onboarding Flow

Use for first-run setup, guided activation, account completion, and feature discovery.

Steps:

1. Explain the immediate value in one clear sentence.
2. Ask only for inputs needed to unlock first success.
3. Show progress, skip rules, and save state.
4. End with a useful first dashboard or next action.

Required states:

- Not started
- In progress
- Completed
- Optional step skipped
- Error and retry

Rules:

- Do not block users with decorative steps.
- Empty states should point back to onboarding setup when needed.
- Motion can guide progress but must not slow completion.
