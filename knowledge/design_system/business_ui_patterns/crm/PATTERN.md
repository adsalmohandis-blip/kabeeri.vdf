# CRM UI Pattern Pack

Use this pack for sales pipelines, account management, support relationships, tasks, and customer activity.

## Design Posture

- Priority: next action, relationship context, pipeline clarity.
- Density: balanced.
- Motion: `BALANCED_MOTION`.
- Dashboard reference: `ADMIT-ADB03` for enterprise shell and relationship data.

## Required Views

- Leads or contacts list
- Pipeline board
- Customer profile
- Tasks
- Activity timeline

## Required Flows

- `CRUD_FLOW.md`
- `USER_MANAGEMENT_FLOW.md`

## Creative Axes

- Kanban versus table emphasis
- Contact profile density
- Activity timeline style
- Quick-add placement
- Sales status vocabulary

## Acceptance Criteria

- Every customer or deal page exposes next action.
- Pipeline movement has feedback and preserves state.
- Activity timeline is scanable and filterable.
- Tasks include due, owner, priority, and completion state.
