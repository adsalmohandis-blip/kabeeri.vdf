# Answer Mapping Rules

## Brand Personality

- Premium, trust, finance, secure: prefer `premium_trust_portal`, `finance_precision`, restrained surfaces.
- Friendly, guided, SaaS, onboarding: prefer `guided_saas_workspace`, `saas_calm`, soft contextual panels.
- Operational, admin, operator: prefer `operator_command_center`, compact density, bordered dense panels.
- Editorial, content, news, docs: prefer `editorial_authority`, comfortable density, readable content bands.
- Commerce, buy, checkout, marketplace: prefer `commerce_conversion_flow`, conversion hierarchy, trust actions.

## Primary User

- Operator/admin: compact, sidebar, data tables, filters, safe actions.
- Executive/manager: KPI-first, balanced density, snapshot cards, data freshness.
- Customer/buyer: decision-first, top navigation, trust signals, clear next action.
- Editor/reader: content-first, table of contents, source list, readable typography.
- Field/mobile worker: action-first, bottom navigation, large touch targets, offline states.

## Risk

- High risk, finance, healthcare, security, government: restrained colors, formal copy, explicit status, low motion.
- Low risk or marketing: more expressive surface style is allowed if performance and accessibility remain intact.

## Language Direction

- Arabic, bilingual, MENA, RTL-first: enable RTL Arabic UI references and verify typography, layout mirroring, forms, tables, numbers, dates, and icon direction.

## Missing Answers

If high-priority answers are missing, ask the missing questions before final visual implementation.
