# Action Label Rules

## Verbs

Use consistent action verbs:

- Create
- Add
- Save
- Update
- Edit
- Delete
- Remove
- Archive
- Restore
- Search
- Filter
- Export
- Import
- Upload
- Download
- Send
- Invite
- Approve
- Reject
- Retry
- Cancel
- View
- Open

## Button Labels

- Primary buttons should describe the completed outcome.
- Destructive actions must name the object when space allows.
- Avoid vague labels such as Submit, OK, Proceed, Continue, or Confirm unless context is unmistakable.
- Repeated actions should keep the same label across the product.
- Icon-only buttons need accessible names and tooltips when meaning is not obvious.

## Examples

- Prefer `Save invoice` over `Submit`.
- Prefer `Delete customer` over `Delete`.
- Prefer `Send invitation` over `Send`.
- Prefer `Apply filters` over `Apply`.
- Prefer `Retry payment` over `Try again` in payment flows.
