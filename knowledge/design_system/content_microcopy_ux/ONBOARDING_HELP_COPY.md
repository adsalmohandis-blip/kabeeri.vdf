# Onboarding And Help Copy

## Onboarding

- Explain one decision per step.
- Show progress when a flow has multiple steps.
- Keep setup copy tied to the user's goal.
- Let users skip non-critical setup when safe.
- Use examples instead of abstract instructions.

## Help Text

- Put help near the decision point.
- Use tooltips for short clarifications, not long instructions.
- Use inline guidance for risky or uncommon choices.
- Use docs links only after giving a short answer in the UI.

## AI Products

- Explain what the AI can and cannot do.
- Tell users when output is generated, streaming, saved, failed, or needs review.
- Provide edit, retry, regenerate, and clear actions.
- Avoid implying certainty when output requires verification.
