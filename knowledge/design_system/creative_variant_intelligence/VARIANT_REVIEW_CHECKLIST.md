# Variant Review Checklist

- Variant preserves required workflow and business logic.
- Variant changes at least four creative axes.
- Variant uses an approved token set or palette preset.
- Variant uses the selected framework adapter.
- Variant uses the selected screen composition or documents why a different composition is needed.
- Variant includes required loading, empty, error, success, disabled, and validation states.
- Variant has a clear audience and tone.
- Variant does not introduce a second UI framework.
- Variant respects accessibility, performance, motion, content, and RTL rules.
- Variant can be implemented from compact references without long prompt repetition.
