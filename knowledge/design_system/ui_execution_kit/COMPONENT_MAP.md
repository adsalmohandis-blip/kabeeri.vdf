# Component Map

Use this matrix to reduce UI decision cost.

## Use Card When

- Grouping related content.
- Displaying page sections.
- Showing dashboard widgets.
- Wrapping a table, form, or chart as a page surface.

## Use Alert When

- Showing persistent success, warning, error, or info message.
- Explaining a blocker that should remain visible.

## Use Toast/Snackbar When

- Showing temporary feedback after a user action.
- Confirming save, delete, export, or import completion.

## Use Badge/Tag When

- Showing status, role, category, count, plan, priority, or risk.

## Use Modal/Dialog When

- Confirming destructive action.
- Editing a small scoped form.
- Asking for a required decision before continuing.

## Use Drawer/Sheet When

- Showing detail preview without leaving a dense table.
- Editing secondary details.
- Mobile navigation.

## Use Table When

- Comparing many rows with similar fields.
- Sorting, filtering, pagination, or bulk actions are required.

## Use Empty State When

- A data list has zero items.
- Search has no results.
- Permission hides all available data.

## Use Chart When

- The user needs trend, comparison, distribution, or progress.
- The chart has a clear question and readable fallback.
