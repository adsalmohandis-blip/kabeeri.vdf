# Adapter Review Checklist

- A single adapter key is selected.
- Package version and install command are recorded.
- Token mapping strategy is referenced.
- Composition mapping is used for layout and components.
- Required imports/build steps are documented.
- Icon strategy is single-source.
- Accessibility behavior is provided by framework components or project contracts.
- RTL and localization requirements are documented when relevant.
- Performance risks from framework CSS/JS/component weight are reviewed.
- Motion uses framework-safe transitions and respects reduced motion.
- No raw design values are hardcoded in page components.
- No second UI framework is introduced without a migration decision.
