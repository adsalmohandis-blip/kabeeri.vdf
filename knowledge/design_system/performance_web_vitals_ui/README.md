# Performance And Core Web Vitals UI Pack

Use this pack for every frontend UI implementation, refactor, design recommendation, or UI review.

The goal is to make performance a design constraint, not a late build optimization. A beautiful UI that shifts layout, delays interaction, ships heavy media, or blocks rendering is not production-ready.

Core order:

1. Choose performance level for the surface.
2. Apply `PERFORMANCE_CONTRACT.md`.
3. Identify LCP, INP, and CLS risks before implementation.
4. Apply image, font, JavaScript, CSS, data rendering, and loading-state rules.
5. Run `PERFORMANCE_REVIEW_CHECKLIST.md` before delivery.

Files:

- `PERFORMANCE_CONTRACT.md`
- `CORE_WEB_VITALS_RULES.md`
- `IMAGE_MEDIA_RULES.md`
- `FONT_CSS_RULES.md`
- `JAVASCRIPT_INTERACTION_RULES.md`
- `DATA_RENDERING_RULES.md`
- `LOADING_SKELETON_RULES.md`
- `PERFORMANCE_REVIEW_CHECKLIST.md`
