# Documentation Generation Lifecycle

This file captures the reusable documentation lifecycle rules extracted from
`KVDF_New_Features_Docs/`.

## Lifecycle Stages

| Stage | Goal | Typical Outputs |
| --- | --- | --- |
| Idea | Capture the project and why it exists. | short brief, intake notes |
| Scope | Define what is included and excluded. | scope statement, boundaries |
| Blueprint | Describe the system structure. | architecture outline, module map |
| Tasking | Break the work into governed execution units. | tasks, ownership, acceptance criteria |
| Execution | Track implementation and docs changes. | progress notes, reports, evidence |
| Validation | Verify the work is correct and complete. | checks, review notes, validation reports |
| Handoff | Deliver the project and summarize the result. | handoff pack, owner summary, release notes |
| Archive | Preserve traceability after delivery. | final reports, logs, history |

## Required Documentation Surfaces

- project overview
- scope and assumptions
- system architecture notes
- task breakdown and acceptance criteria
- validation and review evidence
- handoff and release records
- traceability reports

## Source Package Rule

The generator rules are imported from the source package first, then stored in
permanent Kabeeri folders. The source folder is only a temporary extraction
source and should not remain the long-term home of the rules.
