# New Project Intake

Use this file for projects that start with Kabeeri from day one.

## Required Decisions

| Decision | Required |
| --- | --- |
| Project name | yes |
| Owner | yes |
| Delivery mode | yes |
| Profile: lite, standard, or enterprise | yes |
| Initial stack | optional |
| First release goal | yes |

## Flow

1. Choose delivery mode.
2. Choose profile.
3. Initialize `.kabeeri/project.json`.
4. Create starter folders from the selected generator.
5. Start Structured questionnaires or Agile vision/backlog.
6. Create tasks only after source and acceptance criteria are clear.

## Acceptance Check

A new project intake is valid when the project has a name, owner, selected delivery mode, selected profile, initial source documents, and no execution tasks without acceptance criteria.
