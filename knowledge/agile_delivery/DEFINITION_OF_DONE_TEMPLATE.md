## Done Checklist

- [ ] Acceptance criteria are satisfied.
- [ ] Tests or manual verification evidence are recorded.
- [ ] Empty, loading, error, and permission states are checked when relevant.
- [ ] Security/privacy impact is checked.
- [ ] Migration or data impact is checked when relevant.
- [ ] Design/source governance evidence exists for UI work.
- [ ] AI run history is recorded when meaningful AI assistance was used.
- [ ] ADR is created or linked if the story changed architecture.
- [ ] Handoff notes are updated if client/demo/release behavior changed.
