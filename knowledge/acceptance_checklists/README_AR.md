# قوائم القبول والمراجعة

هذا الفولدر يحدد طبقة قوائم القبول داخل **Kabeeri Vibe Developer Framework**.

الفولدر الحالي بدأ كملف بسيط جدًا، وهذه النسخة تحوله إلى نظام عملي لمراجعة ناتج الذكاء الاصطناعي قبل اعتباره مكتملًا.

## الهدف

قوائم القبول تساعد مطور الذكاء الاصطناعي يجاوب على سؤال مهم:

```text
هل الناتج الذي أنشأه AI جيد بما يكفي لقبوله؟
```

تربط بين:

```text
تعريف المهمة
→ ناتج AI
→ مراجعة بشرية
→ فحص الجودة
→ قرار القبول
→ Commit
→ إغلاق GitHub Issue
```

## ما وظيفة هذا الفولدر؟

يوفر هذا الفولدر قوالب مراجعة من أجل:

- مراجعة الملفات التي أنشأها AI.
- قبول المهمة.
- قبول ناتج فولدر كامل.
- قبول Prompt Pack.
- قبول مجموعة مستندات مولدة.
- قبول نسخة Release Candidate.
- تسجيل ملاحظات المراجعة والقرار النهائي.

## ما الذي لا يفعله؟

هذا الفولدر لا يستبدل:

- الاختبارات البرمجية.
- مراجعة الأمان.
- مراجعة الكود بواسطة خبير.
- اختبار الإنتاج.
- المراجعة القانونية.
- مراجعة الامتثال.

هو فقط نظام Checklists داخل الفريمورك يساعد المبتدئين على عدم قبول ناتج AI بسرعة بدون مراجعة.

## طريقة الاستخدام المقترحة

```text
1. ابدأ من مهمة داخل task_tracking/.
2. نفذ Prompt واحد أو خطوة واحدة مع AI.
3. راجع الملفات التي تغيّرت.
4. استخدم Checklist المناسب.
5. علّم كل بند: pass أو fail أو not_applicable.
6. إذا فشل بند مهم، أعد الناتج إلى AI لإصلاحه.
7. لا تقبل المهمة إلا بعد نجاح القائمة.
8. اعمل Commit واقفل GitHub Issue المرتبطة.
```

## قرارات القبول

```text
accepted
needs_changes
blocked
rejected
not_applicable
```

## أقل قاعدة قبول

لا تعتبر المهمة Done إلا إذا:

```text
- الناتج مطابق لنطاق المهمة.
- لم يتم تعديل ملفات غير مرتبطة.
- لا توجد أسرار حقيقية.
- الناتج مفهوم.
- تم تشغيل الفحوصات المطلوبة أو توثيقها.
- المخاطر المتبقية مكتوبة.
```
