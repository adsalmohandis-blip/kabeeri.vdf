# Task Completion Acceptance Checklist

Use this checklist before closing a GitHub Issue.

## Task linkage

```text
- [ ] Task has a clear title.
- [ ] Task has the correct issue number.
- [ ] Task is assigned to the correct milestone.
- [ ] Task has the correct labels.
- [ ] Task is on the GitHub Project board.
```

## Completion

```text
- [ ] Scope was completed.
- [ ] Acceptance criteria are met.
- [ ] Out-of-scope items were not added.
- [ ] Review notes are written.
- [ ] Remaining risks are documented.
```

## Git

```text
- [ ] `git status` was reviewed.
- [ ] Commit message is clear.
- [ ] Commit was pushed.
- [ ] Closing line was used if needed.
```

## GitHub

```text
- [ ] Project card moved to Done.
- [ ] Issue closed.
- [ ] Milestone progress updated.
```

## Final decision

```text
accepted / needs_changes / blocked / rejected
```
