# Services

Booking flows for services with availability, time slots, confirmations, and service-specific intake.

Intake questions:
- Is this a service catalog, a one-to-one booking, or both?
- Do services have duration, staff assignment, or location constraints?
- Do you need deposits, quotes, or service-specific forms?
- Should the booking route into fulfillment or manual review?

Module map:
- service catalog
- service detail
- slot and calendar picker
- intake form
- confirmation and reminders
- admin service management

Sample acceptance criteria:
- Service details show price, duration, and availability.
- A customer can book a service from start to confirmation.
- Slot rules prevent invalid or overlapping bookings.
- Admin can edit service availability and pricing.

Starter task pack:
- define service catalog and slot rules
- build service detail and booking screens
- wire confirmations and reminders
- create admin tools for service updates
