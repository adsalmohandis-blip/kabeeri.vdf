# Services

Service commerce flows for booking, quoting, deposits, and fulfillment.

Intake questions:
- Is the service sold as a fixed price, quote, or package?
- Do you need service booking and schedule coordination?
- Should deposits, invoices, or staged payments be supported?
- Do you need post-booking fulfillment or follow-up tasks?

Module map:
- service catalog
- quote or estimate flow
- booking or scheduling
- deposit and payment flow
- fulfillment and status updates
- admin service management

Sample acceptance criteria:
- Customers can request or purchase a service.
- Quotes or prices are clearly shown before purchase.
- Deposit or payment state is tracked.
- Admin can manage service availability and fulfillment.

Starter task pack:
- define service catalog and quote rules
- build service request and booking flow
- add deposit/payment handling
- create admin fulfillment controls
