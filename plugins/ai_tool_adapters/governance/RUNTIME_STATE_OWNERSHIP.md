# Runtime State Ownership

`ai_tool_adapter` owns local runtime files that describe registry state,
policy results, evidence, and visibility reports.

## Runtime Files

- `.kabeeri/ai_tool_adapters.json`
- `.kabeeri/ai_tool_runs.jsonl`
- `.kabeeri/ai_tool_policy_results.json`
- `.kabeeri/reports/ai_tool_adapters_dashboard.json`
- `.kabeeri/reports/ai_tool_adapters_readiness.json`

## Ownership Rules

- these files are workspace-local, not source-of-truth artifacts
- they may be regenerated by the plugin
- they must not override repository truth
- they must not become approval state
- they must be treated as compatibility-preserving records, even though the
  public-facing plugin identity is singular

## Compatibility Note

The filenames remain plural for now because the existing implementation and
tests already use them. That is intentional compatibility, not a second plugin.
