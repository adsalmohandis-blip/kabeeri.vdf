# Subscription

Recurring billing and membership-driven ecommerce flows.

Intake questions:
- Is this a membership, subscription box, service plan, or access plan?
- Do you need trials, renewals, upgrades, or cancellations?
- Should users manage billing and plans in their account?
- Do you need access control based on active status?

Module map:
- plans and pricing
- subscription checkout
- billing and renewals
- account and entitlements
- cancellation and upgrade flows
- admin subscriber management

Sample acceptance criteria:
- A customer can start, update, or cancel a subscription.
- Renewal and billing state is tracked.
- Access is gated by active entitlements.
- Admin can manage plans and subscriber status.

Starter task pack:
- define plan rules and billing lifecycle
- build subscription checkout and account screens
- add renewal and cancellation handling
- create admin subscriber management
