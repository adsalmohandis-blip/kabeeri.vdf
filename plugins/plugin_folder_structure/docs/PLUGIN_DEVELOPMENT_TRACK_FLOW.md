# Plugin Development Track Flow

1. Choose `--track=plugin_dev`.
2. Create the plugin workspace in `./workspaces/plugins/<plugin-slug>/`.
3. Collect inputs, identity, plans, specifications, tasks, agents, tests, evidence, and review artifacts.
4. Keep the candidate package in `./workspaces/plugins/<plugin-slug>/04_plugin_package/`.
5. Validate readiness.
6. Request owner review or marketplace upload only when ready.
