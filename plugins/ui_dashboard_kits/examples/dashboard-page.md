# Dashboard Page Recipe

Use for operational overview pages.

## Required Sections

1. Header with title, date/filter controls, and primary action when relevant.
2. KPI cards.
3. Main chart or priority panel.
4. Secondary chart/list/table.
5. Recent activity or alerts.
6. Loading, empty, error, and stale-data states.

## Acceptance Criteria

- KPIs include label, value, trend/status, and accessible meaning.
- Charts have titles and empty/error fallbacks.
- Alerts or blockers are visible before decorative content.
- Role-specific widgets are hidden or explained by permission state.
- Tables/lists are responsive.

## Variation Hooks

Vary these from product answers:

- KPI-first, alert-first, task-first, or chart-first hierarchy;
- dark shell or light shell;
- compact or comfortable density;
- chart type and information order;
- role-specific widget set.
