# Form Page Recipe

Use for create/edit/settings pages.

## Required Sections

1. Header with title and short description.
2. Form card or section groups.
3. Visible labels for every input.
4. Helper text when input meaning is not obvious.
5. Validation states.
6. Save and Cancel actions.
7. Success feedback and error state.

## Acceptance Criteria

- Submit action has Save/Create icon.
- Cancel action has Cancel/Close icon.
- Required fields are clearly marked.
- Error messages are close to fields.
- Submit button disables or shows loading while saving.
- Form remains usable on mobile.

## Variation Hooks

Vary these from product answers:

- single card, section cards, wizard, or two-column form;
- compact enterprise form or spacious public form;
- helper text tone;
- inline validation or summary alert;
- sticky action bar for long forms.
