# Case 4 KCloud Governance

Case 4 is the Kabeeri / KVDOS Cloud governance layer for `multi_ai_governance`.

## Authority

- `multi_ai_governance` is the authority.
- `kcloud_data_sharing` is the transport/sync layer when present.
- `github_provider` is not part of this case.

## What it governs

- Cloud node identity mapping.
- Tenant, organization, project, and local-project mapping.
- Cloud trust and permissions.
- Cloud task tokens.
- Cloud leases.
- Cloud packet and policy checks.
- Cloud audit and evidence records.

## What it does not govern

- IDE window governance.
- Local project governance.
- Wi-Fi/LAN governance.
- GitHub provider behavior.
