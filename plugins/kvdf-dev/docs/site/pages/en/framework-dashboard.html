Framework Dashboard

The framework dashboard is the control surface for KVDF development. It shows core health, governance, boundary state, scorecard status, and the next exact action for the framework owner.

Topic Details

Purpose

The framework dashboard is the control room for Kabeeri development. It gives the framework owner one place to see what the system is doing, where it is healthy, and what still needs attention.

What it shows

The dashboard should surface governance, validation status, boundaries, scorecards, plugin health, docs alignment, and the next exact action. It is the highest-level operational view for framework work.

What it is not

It is not a customer app dashboard and it is not a report archive. Its job is to show the current framework truth, not to collect every historical artifact in one place.

How it uses state

The dashboard should read from canonical workspace and runtime state so the display stays aligned with reality. If the dashboard invents its own state, it becomes a second truth source and loses trust.

Why it matters

A good framework dashboard lets the owner spot drift early and choose the next slice quickly. That is what turns the framework from a collection of files into an actively governed system.

How to use it

When you open the framework dashboard, look for the current boundary, the current risks, the last validation result, and the next action. Those four questions usually tell you where the framework work should go next.
