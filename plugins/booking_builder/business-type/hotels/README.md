# Hotels

Booking flows for rooms, stays, inventory, and reservation management.

Intake questions:
- Do you need room types, rates, and occupancy rules?
- Are there seasonal prices, promos, or packages?
- Do you need add-ons such as breakfast, parking, or transfers?
- Should reservations support modification and cancellation policies?

Module map:
- room catalog
- rate and availability grid
- booking and guest details
- payment or deposit flow
- reservation confirmation
- admin inventory management

Sample acceptance criteria:
- Guests can search rooms by date and occupancy.
- Availability prevents overbooking.
- Rates and add-ons are shown before confirmation.
- Admin can manage inventory and pricing safely.

Starter task pack:
- define room types and rate rules
- build search and availability screens
- add reservation and deposit flow
- create admin inventory controls
