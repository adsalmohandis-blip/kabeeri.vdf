# Technical

Technical blogs are for docs-style posts, tutorials, release notes, and engineering content.

Intake questions:
- Do you need code snippets or doc-style formatting?
- Should tutorials, release notes, or architecture notes be prioritized?
- What taxonomy should organize technical content?

Module map:
- tutorial pages
- release notes
- code snippet support
- docs-style archive
- admin editor

Sample acceptance criteria:
- Code-heavy posts render clearly.
- Readers can browse tutorials and release notes.
- Technical taxonomy is easy to navigate.

Starter task pack:
- define technical content categories
- build docs-style post layouts
- add release-note and tutorial pages
