# Plugin Dev Orchestrator

`plugin_dev` is the plugin-development lifecycle orchestrator.

It consumes workspaces created and validated by `plugin_folder_structure`, then helps with:

- intake
- library analysis
- specifications
- governed tasks
- source builds
- integration contracts
- validation
- evidence
- readiness
- packaging
- promotion requests

It does not create workspace structure directly and it does not install or publish plugins.
