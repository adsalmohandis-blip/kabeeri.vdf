# Agile Delivery

Agile Delivery is the iterative Kabeeri delivery mode. It uses backlog, epics, user stories, sprint planning, sprint review, and sprint cost metadata while keeping the same task governance and acceptance discipline as Structured Delivery.

This canonical v2 file points to the detailed guide in [../agile_delivery/README.md](../agile_delivery/README.md).

## Core Rules

- Agile does not cancel Structured Delivery.
- Agile does not remove documentation; documentation evolves with stories and sprints.
- Every story/task needs a source, acceptance criteria, and review path.
- Sprint cost metadata must track AI tokens, human effort, rework, and accepted output.

## Required Agile Artifacts

- Product backlog
- Epics
- User stories
- Sprint planning notes
- Sprint review notes
- Sprint cost metadata
- Task provenance records
