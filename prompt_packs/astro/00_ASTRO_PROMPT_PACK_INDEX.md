# 00 — Astro Prompt Pack Index

## Framework

Kabeeri Vibe Developer Framework

## Prompt pack

Astro

## Version

v0.1.1

## Goal

Provide a safe, beginner-friendly Astro prompt pack that helps vibe developers build Astro sites step by step after completing Kabeeri VDF planning documents.

## What this pack is

This pack is a set of structured prompts for AI coding assistants.

It helps AI work on Astro projects in a controlled order.

## What this pack is not

This pack is not:

- an Astro installer
- a Node.js installer
- a package manager
- a hosting setup tool
- a full website by itself
- a replacement for official Astro documentation

## Best use cases

Astro is especially suitable for:

```text
Landing pages
Marketing websites
Documentation sites
Blogs
Content websites
Static websites
SEO-focused sites
Hybrid pages with interactive islands
```

## Before using this pack

The user should already have, or should ask AI to help create:

```text
01_STRATEGY_AND_BUSINESS
02_FOUNDATION_ARCHITECTURE
03_RELEASE_SPECS
05_EXECUTION_PLAN
06_ENGINEERING_STANDARDS
08_AI_CODE_PROMPTS
09_TASK_TRACKING
10_ACCEPTANCE_CHECKLISTS
```

For content-heavy projects, also use content/data planning documents when available.

## Prompt order

Use the prompts in this order:

```text
01_PROJECT_CONTEXT_PROMPT.md
02_SITE_STRUCTURE_PROMPT.md
03_ENV_CONFIG_CONTENT_SOURCE_PROMPT.md
04_LAYOUTS_PAGES_NAVIGATION_PROMPT.md
05_COMPONENTS_DESIGN_SYSTEM_PROMPT.md
06_CONTENT_COLLECTIONS_PROMPT.md
07_CMS_INTEGRATION_PROMPT.md
08_FORMS_LEADS_PROMPT.md
09_SEO_METADATA_PERFORMANCE_PROMPT.md
10_INTERACTIVE_ISLANDS_PROMPT.md
11_TESTING_REVIEW_PROMPT.md
12_RELEASE_HANDOFF_PROMPT.md
```

## What each prompt does

| Prompt | Purpose |
|---|---|
| 01 | Give AI the product/site context and first-release scope. |
| 02 | Review or prepare Astro site structure. |
| 03 | Configure environment variables and content source rules. |
| 04 | Add layouts, pages, and navigation foundation. |
| 05 | Add design system and reusable component foundation. |
| 06 | Add content collections for blog/docs/content sites. |
| 07 | Add CMS integration if needed. |
| 08 | Add forms and lead capture foundation if needed. |
| 09 | Add SEO, metadata, and performance review foundation. |
| 10 | Add interactive islands only when needed. |
| 11 | Review tests, quality, accessibility, and performance. |
| 12 | Prepare release handoff summary. |

## Important

These prompts are intentionally small. They should not be merged into one large prompt.
