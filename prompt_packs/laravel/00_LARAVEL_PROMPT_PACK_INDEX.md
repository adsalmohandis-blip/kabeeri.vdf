# 00 — Laravel Prompt Pack Index

## Framework

Kabeeri Vibe Developer Framework

## Prompt pack

Laravel

## Version

v0.1.1

## Goal

Provide a safe, beginner-friendly Laravel prompt pack that helps vibe developers build a Laravel project step by step after completing Kabeeri VDF planning documents.

## Before using this pack

The user should already have, or should ask AI to help create:

```text
01_STRATEGY_AND_BUSINESS
02_FOUNDATION_ARCHITECTURE
03_RELEASE_SPECS
04_DATABASE_ARCHITECTURE
05_EXECUTION_PLAN
06_ENGINEERING_STANDARDS
08_AI_CODE_PROMPTS
09_TASK_TRACKING
10_ACCEPTANCE_CHECKLISTS
```

For small Lite projects, not all folders are required, but the product idea and first release scope should be clear.

## Prompt order

Use the prompts in this order:

```text
01_PROJECT_SETUP_PROMPT.md
02_ENV_AND_DATABASE_PROMPT.md
03_AUTH_USERS_PROMPT.md
04_ROLES_PERMISSIONS_PROMPT.md
05_ADMIN_DASHBOARD_PROMPT.md
06_CORE_MODELS_PROMPT.md
07_SETTINGS_FEATURE_FLAGS_PROMPT.md
08_ACTIVITY_AUDIT_LOGS_PROMPT.md
09_MEDIA_FOUNDATION_PROMPT.md
10_CMS_FOUNDATION_PROMPT.md
11_TESTING_REVIEW_PROMPT.md
12_RELEASE_HANDOFF_PROMPT.md
```

## What each prompt does

| Prompt | Purpose |
|---|---|
| 01 | Create or prepare the Laravel project foundation. |
| 02 | Configure environment, database, queues, cache, mail, and local settings. |
| 03 | Build authentication, users, and profiles foundation. |
| 04 | Build simple roles and permissions foundation. |
| 05 | Build the first admin dashboard foundation. |
| 06 | Create the first core business models based on project documents. |
| 07 | Add settings and feature flags foundation. |
| 08 | Add activity logs and audit logs foundation. |
| 09 | Add media/file upload foundation. |
| 10 | Add basic CMS/content foundation. |
| 11 | Review tests, quality, and safety. |
| 12 | Prepare release handoff summary. |

## Important

These prompts are intentionally small. They should not be merged into one large prompt.
