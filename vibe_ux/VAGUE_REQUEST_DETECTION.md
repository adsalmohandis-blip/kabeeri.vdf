# Vague Request Detection

Vague requests should trigger clarifying questions or safe suggested scopes.

## Vague Signals

- "make it better"
- "fix everything"
- "improve UI"
- "clean the project"
- "make it production ready"
- "add dashboard"
- "connect payments"
- no target user
- no affected surface
- no acceptance criteria
- broad file scope

## Response Rules

- Ask one to three focused questions when the answer changes scope.
- Offer a safe small first task when possible.
- Mark uncertainty in the suggested task card.
- Do not create broad execution tasks from vague text.

## Safe First Task Pattern

When the user asks for a broad improvement, suggest an audit or planning task before implementation.
