# Interaction Modes

Kabeeri supports four interaction modes.

## Vibe Mode

Low-friction mode for natural language requests, free coding, suggested tasks, and post-work capture.

## Guided Mode

Step-by-step mode using v5 adaptive questionnaires and system capability coverage. Best for beginners and project intake.

## Governance Mode

Strict mode for teams, agencies, client work, and sensitive projects. Requires approvals, source, acceptance, locks, budgets, and Owner verify.

## CLI Mode

Advanced mode for automation, CI, scripting, and power users. CLI remains optional for core flows.

## Mode Switching

Mode changes must be logged. Loosening governance on a client, team, production, publish, payment, security, or migration project requires Owner approval.
