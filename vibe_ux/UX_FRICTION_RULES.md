# UX Friction Rules

## Common Action Targets

- Create a suggested task from natural language in one interaction.
- Approve or reject a suggested task from the card.
- Show why a task is blocked without requiring terminal output.
- Estimate AI cost before premium execution.
- Capture changed files after free coding from one action.
- Owner verify from a protected queue.

## Progressive Disclosure

Show simple status first. Reveal raw command details, JSON, audit events, and logs only when the user asks or needs to debug.

## No-Terminal Core Flow

Users should be able to start a project, answer questions, create tasks, approve tasks, run checks, inspect cost, capture work, and verify without manually typing terminal commands.

## Accessibility

Primary surfaces should support keyboard navigation, clear focus, readable contrast, screen-reader labels, plain errors, and RTL-ready layouts for Arabic.
