# Vibe UX Principles

## Guide, Do Not Interrupt

Kabeeri should help users keep momentum. It should surface governance when risk, cost, conflict, verification, or publishing requires attention.

## Suggest, Do Not Force

Natural language requests become suggestions first. Users review, edit, approve, or reject suggested tasks before execution.

## Capture After Work

Free coding is allowed. Post-work capture turns changed files into changesets, suggested tasks, review notes, and acceptance evidence.

## Approval Only When Needed

Approvals are required for high-risk actions such as Owner verify, publish, migration, budget override, owner transfer, broad file scope, and sensitive data sharing.

## CLI Has UI Equivalents

Common CLI commands should map to UI actions. The terminal remains available for power users, automation, and CI, but it is not the default experience.

## Source Of Truth Stays Stable

VS Code, dashboard, chat, and future apps must read and write `.kabeeri/` state through governed commands or equivalent policy rules.
