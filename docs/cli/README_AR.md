# تصميم KVDF CLI

هذا الفولدر يحدد أول تصميم لأوامر أداة سطر الأوامر المستقبلية الخاصة بـ **Kabeeri Vibe Developer Framework**.

## توضيح مهم

هذا الفولدر **ليس تنفيذًا حقيقيًا للـ CLI حتى الآن**.

لا يقوم بتثبيت Packages، ولا تشغيل أوامر فعلية، ولا توليد مشاريع حقيقية، ولا تعديل الريبو تلقائيًا.

هو فقط طبقة تصميم ومواصفات للأداة المستقبلية.

## الاسم المقترح للـ CLI

```text
kvdf
```

أسماء بديلة للحزمة أو العرض:

```text
kabeeri-vdf
kabeeri.vdf
```

## الهدف

الـ CLI المستقبلي يساعد مطور الذكاء الاصطناعي على استخدام Kabeeri VDF من الطرفية بطريقة بسيطة ومنظمة.

يساعد في:

```text
- إنشاء Workspace خاص بـ Kabeeri VDF
- اختيار Lite أو Standard أو Enterprise
- نسخ القوالب
- إنشاء ملفات الأسئلة
- اختيار Prompt Packs
- إنشاء ملفات Task Tracking
- إنشاء Acceptance Checklists
- فحص ملفات الفريمورك
- تجهيز ملفات Release Handoff
```

## ما الذي لا يجب أن يفعله في النسخ الأولى؟

في النسخ الأولى، لا يجب أن يقوم الـ CLI بـ:

```text
- تثبيت Laravel أو .NET أو Next.js أو أي فريمورك آخر
- إنشاء تطبيق إنتاج كامل تلقائيًا
- نشر المشاريع
- تعديل أنظمة حية
- حفظ أسرار حقيقية
- استبدال GitHub Issues
- تشغيل أوامر خطيرة أو مدمرة
```

## فلسفة الأداة

الـ CLI ليس أداة سحرية لبناء التطبيق بالكامل.

هو مساعد آمن لتجهيز ملفات Kabeeri VDF بشكل منظم.

```text
فكرة خام
→ اختيار profile
→ توليد مستندات الفريمورك
→ اختيار prompt pack
→ إنشاء مهام
→ مراجعة بقوائم القبول
→ تسليم
```

## نطاق النسخة الحالية

في `v0.1.1` المطلوب هو تصميم الأوامر فقط.

التنفيذ الفعلي يمكن أن يبدأ في نسخة لاحقة مثل `v0.2.0`.

## عائلة الأوامر المقترحة

```text
kvdf init
kvdf generate
kvdf questionnaire
kvdf prompt-pack
kvdf task
kvdf acceptance
kvdf example
kvdf validate
kvdf release
kvdf doctor
```

## لغة التنفيذ المقترحة مستقبلًا

الأفضل كبداية:

```text
Node.js / TypeScript
```

لأنها مناسبة للتوزيع عبر npm وسهلة للتعامل مع JSON وMarkdown.

## الحالة

تصميم هيكل أوامر CLI للنسخة `v0.1.1`.
