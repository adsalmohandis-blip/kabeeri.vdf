# 01 — Vision and Positioning

## Official name

**Kabeeri Vibe Developer Framework**  
Short name: **Kabeeri VDF**  
Repository name: **kabeeri.vdf**

## Short definition

Kabeeri VDF is an open-source meta-framework for AI-driven software development. It helps vibe developers turn raw product ideas into structured folders, beginner-friendly questionnaires, AI-ready documents, coding prompts, implementation tasks, and acceptance checklists before writing code with Laravel, .NET, Next.js, WordPress, Django, or any other stack.

## Mission

To help AI-powered builders create software in a structured, repeatable, reviewable, and beginner-friendly way.

Kabeeri VDF exists because many people can now ask AI tools to write code, but they still struggle to define the product, organize the work, separate core features from future ideas, track what AI produced, and review the result safely.

## The core problem

AI coding tools can generate code quickly, but they do not automatically solve product thinking, architecture, planning, scoping, task tracking, or acceptance review.

Common problems include:

- starting with a vague prompt such as “build me an app”
- generating code before the product is clear
- mixing the first version with future features
- producing many files without a clear system
- losing track of what has been completed
- accepting AI output without review
- rebuilding the same planning process for every project

Kabeeri VDF provides a repeatable development workflow for these problems.

## Positioning

Kabeeri VDF is **not** a replacement for Laravel, .NET, Next.js, Django, WordPress, or any implementation framework.

It is a layer above them.

```text
Raw idea
→ Kabeeri VDF
→ AI-ready documents and prompts
→ Laravel / .NET / Next.js / WordPress / other stack
→ code
→ review
→ release
```

A traditional code framework helps developers write code faster. Kabeeri VDF helps vibe developers know what to ask AI, in what order, and how to review the result.

## Target users

Kabeeri VDF is designed for:

- vibe developers
- AI-powered software builders
- founders building software with AI
- product owners who need structured planning
- small teams using ChatGPT, Codex, Cursor, Claude Code, Windsurf, GitHub Copilot, or similar tools
- agencies that want a repeatable AI delivery workflow
- traditional developers who want a planning layer before code generation

## What Kabeeri VDF is not

Kabeeri VDF is not:

- a no-code platform
- a low-code platform
- only a prompt library
- only a folder template
- only a code generator
- a replacement for engineering review
- a guarantee that any idea becomes a successful product

It is a framework for organizing the work before, during, and after AI-assisted code generation.

## Core workflow

```text
Idea
→ Project profile
→ Skeleton generation
→ Architecture guide
→ Folder questionnaires
→ Folder documents
→ Prompt packs
→ Task tracking
→ AI coding
→ Acceptance review
→ Extension planning
```

## Product profiles

Kabeeri VDF supports three planned profiles.

### Lite

For small MVPs, landing pages, simple dashboards, internal tools, and early product experiments.

### Standard

For SaaS products, web apps, CMS-style products, booking systems, e-commerce systems, and typical business applications.

### Enterprise

For large platforms, multi-tenant systems, marketplaces, ERP-style systems, AI platforms, and long-term product ecosystems.

## Guiding principles

1. **Start before code** — understand the product before generating implementation.
2. **Use beginner-friendly questions** — do not require deep technical language from the project owner.
3. **Generate documents from answers** — avoid empty documents that look complete but contain no project truth.
4. **Use one prompt for one task** — keep AI coding tasks small and reviewable.
5. **Track everything** — every AI task should have a status and review path.
6. **Review before accepting** — AI output should be checked against acceptance criteria.
7. **Protect the core** — future ideas should go through the extension layer, not directly into the core project.

## Success vision

A successful Kabeeri VDF project should allow a user to:

- explain a product idea clearly
- generate a clean project skeleton
- answer folder-specific questions
- produce useful project documents with AI
- generate safe and focused coding prompts
- track implementation tasks
- review completed work
- plan future extensions without damaging the core system
