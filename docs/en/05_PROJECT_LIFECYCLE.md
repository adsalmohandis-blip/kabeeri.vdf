# 05 - Project Lifecycle

## Stage 1 - Create Skeleton

The owner chooses a Lite, Standard, or Enterprise profile. Kabeeri creates the
project skeleton and the first structured files.

## Stage 2 - Answer Questions

The owner answers the essential project questions first: strategy, architecture,
release scope, data/API needs, and execution plan.

## Stage 3 - Generate Folder Documents

Answered questionnaires can be used to generate focused documents for specific
folders or system areas.

## Stage 4 - Choose Implementation Stack

After planning is clear, the team chooses the implementation stack: Laravel,
Next.js, React Native Expo, Flutter, .NET, Django, WordPress, or another pack.

## Stage 5 - Select Prompt Pack

The matching prompt pack provides small AI implementation prompts for the chosen
stack.

## Stage 6 - Execute Governed Tasks

Each prompt should map to a governed task with allowed files, forbidden files,
acceptance criteria, and review expectations.

## Stage 7 - Review and Accept

Work is not complete until it passes review, policy gates, and acceptance
criteria.

## Stage 8 - Prepare Release

Release work includes QA, security, documentation, migration safety, handoff,
and publish gates.

## Stage 9 - Continue Through Extension Layer

Future capabilities should start as extension-layer work and become core only
after they are stable.
