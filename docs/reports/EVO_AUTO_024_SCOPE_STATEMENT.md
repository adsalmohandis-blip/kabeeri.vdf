# EVO_AUTO_024 Scope Statement

## Priority

- ID: `evo-auto-024-quality-gates`
- Title: `Quality Gates`
- Source: `new_features_docs_study`

## Scope

This priority keeps readiness and done gates explicit before work advances or closes.

The scope includes:

- readiness gating
- done gating
- blocked visibility
- validation-aligned lifecycle behavior

## Out of scope

- collapsing the gate model into a simple open/closed flag
- removing blocked states from the lifecycle
- changing unrelated task planning behavior

## Success criteria

- readiness remains explicit
- done remains distinct from ready
- blocked states are preserved
- downstream governance can rely on the gate model without ambiguity
