# 15 — بحث السوق والتميّز

## ملاحظة

توجد أدوات ومشاريع قريبة من الفكرة، لكنها غالبًا تركز على AI agents أو spec-driven development أو coding داخل IDE. Kabeeri VDF يتميز بأنه موجه للـ vibe developer المبتدئ، ويبدأ من أسئلة فولدرات واضحة قبل الكود.

## أدوات ومشاريع قريبة

### Kiro

Agentic IDE يستخدم spec-driven development لتحويل prompts إلى specs ثم code/docs/tests. قريب من فكرة الانتقال من prompt عشوائي إلى مواصفات.

### GitHub Spec Kit

Toolkit مفتوح المصدر لـ spec-driven development. يركز على جعل المواصفات قابلة للتنفيذ مع agents.

### MetaGPT

Multi-agent framework يحاكي شركة برمجيات بأدوار مثل Product Manager و Architect و Engineer، ويخرج PRD وتصميم ومهام وكود.

### OpenHands

منصة open-source لـ AI software agents تستطيع تنفيذ أعمال هندسية، تعديل كود، وتشغيل أوامر.

### GitHub Copilot Coding Agent

يمكنه العمل على Issues وفتح Pull Requests وتشغيل اختبارات وطلب مراجعة.

## الفرق المقترح لـ Kabeeri VDF

```text
Kiro / Spec Kit = Spec-driven coding workflow
MetaGPT = multi-agent software company simulation
OpenHands = autonomous coding agent platform
Copilot Agent = code execution inside GitHub workflow
Kabeeri VDF = beginner-friendly AI development meta-framework based on folders, questionnaires, documents, prompts, tasks, and acceptance
```

## مساحة التميز

- موجه للمبتدئين وليس فقط المهندسين.
- عربي/إنجليزي من البداية.
- يعتمد على Folder Questionnaires.
- يفصل Skeleton عن توليد المحتوى.
- يدعم Core / Production / Extension layers.
- قابل للاستخدام مع أي AI chat وليس أداة واحدة فقط.
- يمكن أن يتحول إلى CLI و VS Code و Desktop و SaaS تدريجيًا.

## الاستنتاج

لا تبدأ بتقديمه كـ “بديل لـ Kiro أو GitHub Spec Kit”، بل كـ Framework عملي للـ vibe developer فوق كل هذه الأدوات.
