# 12 — Extension Layer

## الهدف

حماية النظام الأساسي من المميزات المستقبلية غير المستقرة.

## الفرق بين Core و Extension

```text
Core = النظام لا يعمل بدونه.
Extension = ميزة يمكن إضافتها أو حذفها دون كسر الأساس.
Experimental = فكرة تحت التجربة.
```

## قواعد الإضافات

- أي ميزة بعد الإطلاق تبدأ كـ Extension Proposal.
- لا يتم تعديل Core مباشرة.
- يجب تحديد تأثير الإضافة على البيانات والصلاحيات والأسعار والواجهة.
- يجب أن يكون لكل إضافة Prompt Pack و Acceptance Checklist.
