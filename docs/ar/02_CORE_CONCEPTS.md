# 02 — المفاهيم الأساسية

## Meta-Framework

Kabeeri VDF ليس Framework للكود فقط، بل Framework فوق أطر العمل. هو ينظم التفكير والتوثيق والبرومبتات والتتبع والمراجعة.

## Vibe Developer

هو شخص يستخدم الذكاء الاصطناعي لبناء البرمجيات، وقد لا يكون مبرمجًا محترفًا. يحتاج إلى لغة سهلة، أسئلة واضحة، وخطوات تنفيذ مرتبة.

## Skeleton

هو هيكل المشروع الأساسي: فولدرات + دليل معمارية + ملف أسئلة داخل كل فولدر. لا يحتوي على ملفات تفصيلية كثيرة فارغة.

## Folder Questionnaire

ملف أسئلة داخل كل فولدر. يشرح أهمية الفولدر، نوع الملفات التي سيولدها AI، ثم يسأل صاحب المشروع أسئلة سهلة عن المنتج. بعد الإجابة، يتم إرسال الملف إلى AI لتوليد ملفات هذا الفولدر فقط.

## Prompt Pack

مجموعة برومبتات مخصصة لإطار عمل معين مثل Laravel أو .NET أو Next.js. كل Prompt ينفذ مهمة واحدة أو مجموعة صغيرة قابلة للمراجعة.

## Task Tracking

نظام يتابع حالة كل مهمة: pending, in_progress, ai_done, review_needed, verified, blocked.

## Acceptance Checklist

قائمة مراجعة تؤكد أن الناتج ليس فقط موجودًا، بل صحيح ومفهوم وآمن وقابل للاستخدام.

## Extension Layer

طبقة مخصصة للمميزات المستقبلية بعد إطلاق النسخة الأساسية، حتى لا يتم تلويث الفولدرات الرئيسية بأفكار غير مستقرة.
