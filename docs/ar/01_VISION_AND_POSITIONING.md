# 01 — الرؤية والتموضع

## ما هو Kabeeri Vibe Developer Framework؟

هو إطار عمل فوقي لتطوير البرمجيات بالذكاء الاصطناعي. يعمل قبل وأثناء استخدام أدوات الكود مثل Laravel و .NET و Next.js. الهدف ليس كتابة الكود مباشرة، بل تجهيز صاحب المشروع والذكاء الاصطناعي لفهم المنتج، تقسيمه، توثيقه، تنفيذه، ومراجعته.

## لماذا بدأ من Kabeeri؟

كما خرج React من احتياج داخلي داخل Facebook ثم أصبح إطارًا عامًا، يمكن لـ Kabeeri Vibe Developer Framework أن يبدأ من احتياج منصة Kabeeri ثم يتحول إلى إطار عام يخدم أي شخص يبني بالذكاء الاصطناعي.

## المشكلة التي يحلها

الـ vibe developer غالبًا يعرف الفكرة العامة لكنه لا يعرف:

- ما الملفات التي يحتاجها المشروع.
- كيف يشرح المنتج للذكاء الاصطناعي.
- كيف يقسم النظام إلى مراحل.
- كيف يعرف أن AI خرج عن المطلوب.
- كيف يحافظ على قابلية التطوير.
- كيف ينشئ Extensions بعد الإطلاق بدون كسر الأساس.

## القيمة الأساسية

الفريمورك يحول البرمجة بالذكاء الاصطناعي من محادثات عشوائية إلى دورة عمل منظمة:

```text
Idea → Questionnaire → Documentation → Prompt Pack → Tasks → Code → Acceptance → Extensions
```

## الجمهور المستهدف

- Vibe Developers.
- أصحاب المشاريع.
- فرق صغيرة تبني منتجات بالذكاء الاصطناعي.
- وكالات تطوير تريد طريقة تسليم موحدة.
- مطورون محترفون يريدون تقليل فوضى AI.

## الجملة التسويقية

**From raw idea to AI-buildable software system.**

بالعربي:

**من فكرة خام إلى نظام برمجي قابل للبناء بالذكاء الاصطناعي.**
