# Folder Output Acceptance Checklist

Use this checklist when accepting a full framework folder output.

Examples:

```text
task_tracking/
acceptance_checklists/
prompt_packs/laravel/
questionnaires/core/
```

## Folder purpose

```text
- [ ] Folder purpose is clear.
- [ ] README exists.
- [ ] Files match the folder purpose.
- [ ] Unnecessary files are not present.
```

## File set

```text
- [ ] Main README exists.
- [ ] Arabic README exists when useful.
- [ ] Templates exist if needed.
- [ ] Schema/manifest exists if useful.
- [ ] Example file exists if useful.
```

## Consistency

```text
- [ ] Naming style is consistent.
- [ ] File order is understandable.
- [ ] Terms match Kabeeri VDF terminology.
- [ ] The folder can be used by beginners.
```

## AI usability

```text
- [ ] Instructions are AI-ready.
- [ ] AI scope rules are clear.
- [ ] Allowed/forbidden work areas are clear where needed.
- [ ] The folder helps avoid vague prompts.
```

## Decision

```text
accepted / needs_changes / blocked / rejected
```
