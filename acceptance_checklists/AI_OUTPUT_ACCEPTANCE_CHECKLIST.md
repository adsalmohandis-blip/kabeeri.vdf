# AI Output Acceptance Checklist

Use this checklist after AI creates or edits files.

## Scope

```text
- [ ] AI followed the exact prompt.
- [ ] AI did not add future features.
- [ ] AI did not modify unrelated folders.
- [ ] AI stopped after completing the requested task.
```

## Files

```text
- [ ] Changed files are expected.
- [ ] New files are in the correct folder.
- [ ] No temporary files were added.
- [ ] No duplicate/near-duplicate files were added.
```

## Content/code quality

```text
- [ ] Output is understandable.
- [ ] Output is beginner-friendly where applicable.
- [ ] Naming is clear.
- [ ] The structure matches the framework style.
- [ ] There are no obvious contradictions.
```

## Safety

```text
- [ ] No real secrets.
- [ ] No private keys.
- [ ] No production credentials.
- [ ] No private user data.
- [ ] No dangerous instructions.
```

## Checks

```text
- [ ] Manual review completed.
- [ ] `git status` reviewed.
- [ ] Tests/checks run if relevant.
- [ ] Known issues documented.
```
