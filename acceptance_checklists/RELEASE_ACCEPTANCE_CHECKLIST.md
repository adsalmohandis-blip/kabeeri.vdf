# Release Acceptance Checklist

Use this checklist before publishing a GitHub Release.

## Milestone

```text
- [ ] Milestone exists.
- [ ] All required issues are closed.
- [ ] Project board reflects completion.
- [ ] Open issues are intentionally moved to another milestone.
```

## Repository

```text
- [ ] README is updated.
- [ ] Important folders have README files.
- [ ] New files are committed and pushed.
- [ ] No temporary files are committed.
- [ ] No secrets are committed.
```

## Quality

```text
- [ ] Main examples work or are documented.
- [ ] Schemas/examples are valid JSON where applicable.
- [ ] Links and folder names are consistent.
- [ ] Release notes are ready.
```

## GitHub release

```text
- [ ] Tag name is correct.
- [ ] Release title is correct.
- [ ] Release notes explain what changed.
- [ ] Known limitations are listed.
```

## Decision

```text
accepted / needs_changes / blocked / rejected
```
