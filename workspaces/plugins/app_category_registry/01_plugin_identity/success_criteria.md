# Success Criteria

The plugin is successful when:

- categories are data-driven
- active ready categories are shown by default
- inactive or incomplete categories are hidden or warning-labeled
- selected profiles are deterministic
- conflicts are recorded and surfaced
- questionnaires ask missing details first
- docs and roadmap contracts are resolved from the selected profile
- workspace plans are generated safely
- evidence is written for each major stage
- the plugin never builds the app directly

If these conditions fail, the plugin is not ready for production use.
