# Token Budget Rules

Token budgets define allowed AI usage by task, sprint, workstream, developer/agent, and project.

## Budget Levels

- task budget
- sprint budget
- workstream budget
- developer or AI agent budget
- project budget
- exploration or learning budget

## Thresholds

- 75 percent used: warning
- 90 percent used: acknowledgement required
- 100 percent used: block official usage unless approval exists

## Over-Budget Rules

Over-budget execution requires:

- Owner or Maintainer approval based on policy
- reason for continuing
- expected value of the extra spend
- link to task, sprint, or exploration budget
- audit event

## Reporting

Reports must distinguish:

- accepted output cost
- rejected output cost
- rework cost
- exploration cost
- untracked cost
- urgent incident cost
